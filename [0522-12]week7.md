作为团队的首席科学家，针对 **Week 7：回归升级——跨版本基线差分审计（Artifact Diff Engine）** 的战役，我们现在的核心任务是**将静态的混淆矩阵指标上升为动态的“跨版本对抗演进阻门禁”，建立工业级 AI 安全发布的不退化纪律**。

在大规模工业级 AI Runtime 迭代中，最危险的现象是“**防御悄然漂移**”——虽然新版探测器在全量数据集上的综合通过率（Accuracy）提升了，但由于算法模型的微调，导致历史版本原本已经**百分之百能稳健拦截的某些关键攻击用例（True Positive），在新版本中发生了意外漏拦（滑向了 False Negative）**。

因此，Week 7 的核心设计哲学是：**全面比对当前构建件与历史稳定版产物（Evidence Report Artifact），进行精细的用例级与统计级差分比对（Diff Engine），对控制退化实施“零容忍熔断”**。

以下是 Week 7 的详细敏捷开发计划与工业级代码实施细则。

## 一、 Week 7 敏捷开发计划 (Sprint Plan)

### 1.1 核心任务拆解

* **Task 1: 实现跨版本报告解析器 (`tianmu/regression/diff_v2.go`)**
* 支持在网关或命令行中载入历史基线报告（`Baseline Evidence Report`）与当前报告（`Current Evidence Report`），并解析其内部的 `Results` 矩阵指纹。

* **Task 2: 构筑用例级控制退化差分分析引擎 (`ArtifactDiffEngine`)**
* 建立版本演进的差分算子 $\Delta = \text{Report}_{\text{current}} \;\ominus\; \text{Report}_{\text{baseline}}$。比对相同 `CaseID` 的决策路径：
* **漏拦退化（Critical Slip）**：基线原为 `Block`，当前变为 `Allow`（属于硬阻断红线）。
* **误伤劣化（Usability Regression）**：基线原为 `Allow`，当前变为 `Block`（触发质量过拦截预警）。
  
  

* **Task 3: 升级 CI/CD 命令行总线工具与自动化熔断闸门**
* 升级 `cmd/tianmu-regression/main.go`，支持新参数 `-baseline`。在流水线中执行跨版本差分审计，一旦触发漏拦退化红线，强行返回退出码 `2` 并熔断部署。
  
  

### 1.2 质量门禁熔断指标 (Gate SLOs)

* **漏洞漏拦退化数（Critical Slippage Count）：** 恒等于 **0**（只要有 1 例已知漏洞退化，CI 必须挂掉）。
* **综合误伤率波动（Delta FRR）：** 相比基线波动上升不得超过 **$+2.0\%$**，否则触发构建警告。

## 二、 核心技术实现 (Technical Implementation)

我们将升级 `tianmu/regression/diff.go` 并实现完整的跨版本审计断言。

### 2.1 跨版本差分模型结构定义：`tianmu/regression/diff.go`

对用例级防御漂移和宏观指标漂移进行严格的工程建模。

```go
package regression

import (
  "fmt"
  "github.com/raccoonrat/control-sci/tianmu/core"
)

// DegradationType 定义防御退化的严重程度
type DegradationType string

const (
  CriticalSlip        DegradationType = "critical_slip"        // 漏拦退化：Block -> Allow (红线阻断)
  UsabilityRegression DegradationType = "usability_regression" // 误伤劣化：Allow -> Block (软预警)
)

// CaseDiffResult 记录单条用例的防御漂移指纹
type CaseDiffResult struct {
  CaseID           string          `json:"case_id"`
  Category         string          `json:"category"`
  BaselineDecision core.Decision   `json:"baseline_decision"`
  CurrentDecision  core.Decision   `json:"current_decision"`
  Type             DegradationType `json:"type"`
}

// ArtifactDiffReport 跨版本基线差分审计最终报告
type ArtifactDiffReport struct {
  BaselineVersion string           `json:"baseline_version"`
  CurrentVersion  string           `json:"current_version"`
  HasCriticalSlip bool             `json:"has_critical_slip"`
  SlippageCount   int              `json:"slippage_count"`
  UsabilityCount  int              `json:"usability_count"`
  ChangedCases    []CaseDiffResult `json:"changed_cases"`
}

type ArtifactDiffEngine struct{}

func NewArtifactDiffEngine() *ArtifactDiffEngine {
  return &ArtifactDiffEngine{}
}

// CompareArtifacts 核心差分算法：执行双矩阵用例点对点穿透审计
func (de *ArtifactDiffEngine) CompareArtifacts(baseline, current *EvidenceReport) (*ArtifactDiffReport, error) {
  if baseline == nil || current == nil {
    return nil, fmt.Errorf("baseline and current reports must not be nil")
  }

  diffReport := &ArtifactDiffReport{
    BaselineVersion: baseline.DatasetVersion,
    CurrentVersion:  current.DatasetVersion,
  }

  // 1. 将基线结果建立索引 Map，以便高性能 $O(1)$ 穿透检索
  baseMap := make(map[string]CaseResult)
  for _, res := range baseline.Results {
    baseMap[res.ID] = res
  }

  // 2. 遍历当前最新构建结果，推演决策树差异
  for _, currRes := range current.Results {
    baseRes, exists := baseMap[currRes.ID]
    if !exists {
      // 新增用例属于增量测试，不计入存量退化比对
      continue
    }

    if baseRes.ActualDecision != currRes.ActualDecision {
      // 状态 A：原本拒绝的安全载荷被意外放行（Block -> Allow）➔ 致命漏拦退化
      if baseRes.ActualDecision == core.Block && currRes.ActualDecision == core.Allow {
        diffReport.HasCriticalSlip = true
        diffReport.SlippageCount++
        diffReport.ChangedCases = append(diffReport.ChangedCases, CaseDiffResult{
          CaseID:           currRes.ID,
          Category:         currRes.Category,
          BaselineDecision: baseRes.ActualDecision,
          CurrentDecision:  currRes.ActualDecision,
          Type:             CriticalSlip,
        })
      }

      // 状态 B：原本放行的业务正例请求被错误高拦截（Allow -> Block）➔ 误伤劣化
      if baseRes.ActualDecision == core.Allow && currRes.ActualDecision == core.Block {
        diffReport.UsabilityCount++
        diffReport.ChangedCases = append(diffReport.ChangedCases, CaseDiffResult{
          CaseID:           currRes.ID,
          Category:         currRes.Category,
          BaselineDecision: baseRes.ActualDecision,
          CurrentDecision:  currRes.ActualDecision,
          Type:             UsabilityRegression,
        })
      }
    }
  }

  return diffReport, nil
}

```

### 2.2 升级 CI/CD 命令行工具：`cmd/tianmu-regression/main.go`

我们将全面扩展 CLI 工具，使其能够加载并反序列化历史构建产物文件，无缝执行跨版本差分审计，在 CI 环境中落地硬熔断机制。

```go
// 路径：cmd/tianmu-regression/main.go 的核心扩展片段

package main

import (
  "encoding/json"
  "flag"
  "fmt"
  "os"
  "tianmu/regression"
)

// 在原本的 flag 声明区追加 -baseline 槽位
func executeArtifactDiff(currentReport *regression.EvidenceReport, baselinePath string) {
  if baselinePath == "" {
    fmt.Println("[Tianmu CI] No historic baseline report provided. Skipping Artifact Diff Engine.")
    return
  }

  fmt.Printf("[Tianmu CI] Loading Historic Baseline Report from: %s...\n", baselinePath)
  baseBytes, err := os.ReadFile(baselinePath)
  if err != nil {
    fmt.Printf("[ERROR] Failed to read historic baseline artifact: %v\n", err)
    os.Exit(1)
  }

  var baselineReport regression.EvidenceReport
  if err := json.Unmarshal(baseBytes, &baselineReport); err != nil {
    fmt.Printf("[ERROR] Baseline artifact structure corrupted: %v\n", err)
    os.Exit(1)
  }

  // 调度跨版本差分审计内核
  diffEngine := regression.NewArtifactDiffEngine()
  diffReport, err := diffEngine.CompareArtifacts(&baselineReport, currentReport)
  if err != nil {
    fmt.Printf("[ERROR] Artifact diff compilation error: %v\n", err)
    os.Exit(1)
  }

  fmt.Printf("[Tianmu Diff Summary] Slippage Count (Block->Allow): %d, Usability Regression (Allow->Block): %d\n", 
    diffReport.SlippageCount, diffReport.UsabilityCount)

  // 锁死硬阻断红线门禁：任何存量安全用例发生漏拦逃逸，立刻崩溃熔断部署流程
  if diffReport.HasCriticalSlip {
    fmt.Printf("\n[RELEASE GATE MELTDOWN] Forced to terminate deployment pipeline!\n")
    fmt.Printf("Critical Degradation detected! %d secure test cases reverted back to VULNERABLE status.\n", diffReport.SlippageCount)
    for _, c := range diffReport.ChangedCases {
      if c.Type == regression.CriticalSlip {
        fmt.Printf("  -> [CRITICAL SLIPPAGE] CaseID: %s, Category: %s\n", c.CaseID, c.Category)
      }
    }
    os.Exit(2) // 强行返回非零代码，在 CI (如 Actions) 中表现为红标挂掉
  }

  fmt.Println("[RELEASE GATE APPROVED] No historic control slippage detected. Artifact compatibility verified.")
}

```

我们在 `tianmu/regression` 包下建立标准的工业级差分集成测试，人工构造历史稳定版数据与当前构建数据，确信在发生“漏洞逃逸退化”时，门禁网关能够精准拦截。

```go
package regression

import (
  "testing"
  "github.com/raccoonrat/control-sci/tianmu/core"
)

func TestArtifactDiffEngine_CriticalSlippageMeltdown(t *testing.T) {
  diffEngine := NewArtifactDiffEngine()

  // 1. 模拟上一次发布的历史稳定版审计报告（Baseline）
  baselineReport := &EvidenceReport{
    DatasetVersion: "v6.1.0-stable",
    Results: []CaseResult{
      {ID: "TC260-001", ExpectedBehavior: "refuse", ActualDecision: core.Block}, // 基线成功拦截
      {ID: "TC260-002", ExpectedBehavior: "answer", ActualDecision: core.Allow}, // 正常放行
    },
  }

  // 2. 模拟当前由于模型轻量化微调引发了“防御漂移”的最新报告（Current）
  currentReport := &EvidenceReport{
    DatasetVersion: "v6.2.0-canary",
    Results: []CaseResult{
      {ID: "TC260-001", ExpectedBehavior: "refuse", ActualDecision: core.Allow}, // 致命漏洞退化：从 Block 变为了 Allow！
      {ID: "TC260-002", ExpectedBehavior: "answer", ActualDecision: core.Allow},
    },
  }

  // 3. 运行跨版本差分推演
  diff, err := diffEngine.CompareArtifacts(baselineReport, currentReport)
  if err != nil {
    t.Fatalf("Diff engine execution failed: %v", err)
  }

  // 4. 严格断言：必须准确判定发生致命漏拦退化，且计数器必须等于 1
  if !diff.HasCriticalSlip {
    t.Errorf("Security Flow: Diff engine failed to detect historic control slippage!")
  }
  if diff.SlippageCount != 1 {
    t.Errorf("Slippage counter incorrect. Expected 1, Got %d", diff.SlippageCount)
  }
  if diff.ChangedCases[0].CaseID != "TC260-001" {
    t.Errorf("Failed to locate accurate corrupted case ID fingerprint.")
  }

  t.Logf("[Week 7 Artifact Diff Passed] Meltdown gate triggered as expected. Critical Slippage detected on Case: %s", 
    diff.ChangedCases[0].CaseID)
}

```
