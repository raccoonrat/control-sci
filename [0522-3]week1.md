一期工程（Foundation Phase）**Week 1** 的核心战役正式打响。本周的战略目标极为明确：**确立大模型运行时的“第一宪章”，即冻结核心契约（Control Decision Contract v1），并实现高性能、零拷贝的 Go 语言核心拦截网关骨架。**

这是决定 Tianmu 底座上限的关键一步。为了确保 Personal AI 阶段能以微秒级的开销运行，同时在底层为 Enterprise AI 的多租户、复杂 Action 治理留足扩展槽位（Slots），我们拒绝使用重型的企业级网关中间件，而是直接采用原生的 Go 语言构筑这套高性能运行时控制平面。

## 一、 Week 1 敏捷开发计划 (Sprint Plan)

### 1.1 核心任务拆解

* **Task 1: 契约规范形式化落地**
* 将控制决策函数映射为 Go 语言强类型结构体，固定 I/O 上下文、Actor 身份、资产分级与风险信号的字段标准。

* **Task 2: 零拷贝核心拦截网关（Tianmu Engine Core）开发**
* 构建不可旁路（Non-Bypassable）的中间件/拦截器骨架，作为 AI Runtime 的安全隔离底座。

* **Task 3: 策略评估器（Policy Evaluator）单体原型实现**
* 实现一个基于配置文件的策略加载器，支持根据不同置信度和风险信号，动态收敛出 `Redact`、`Block` 或 `Allow` 决策。

* **Task 4: Release Evidence Lite 自动化生成测试**
* 确保每一次控制决策都能输出标准、无污染且可串联的分布式追踪证据块。
  
  

### 1.2 关键时延预算 (Latency Budget)

* **同步快线检测（Context Parser + Policy Matrix）：** $\le 2\text{ms}$
* **内存序列化与证据块生成：** $\le 0.5\text{ms}$

### 2.1 `tianmu/core/contract.go`

用于定义不可变控制属性、上下文槽位以及可审计的证据链结构。

```go
package core

import (
  "time"
)

type ReleaseStage string
const (
  PersonalAI   ReleaseStage = "personal_ai"
  EnterpriseAI ReleaseStage = "enterprise_ai"

type Decision string
const (
  Allow           Decision = "allow"
  Block           Decision = "block"

  LogOnly         Decision = "log_only"

// RequestContext 环境上下文
```

### 2.2 `tianmu/core/evaluator.go`

实现代数决策矩阵判定，消灭硬编码。它负责将不确定的风险信号，根据预设的 Policy Pack 规则，转化为确定性的控制指令。

```go
package core

import (
  "errors"
)

type PolicyRule struct {
  RiskCategory      string   `json:"risk_category"`
  ConfidenceThresh  float64  `json:"confidence_threshold"`
  TargetDecision    Decision `json:"target_decision"`
  ReasonCode        string   `json:"reason_code"`
}

type PolicyPack struct {
  Version string       `json:"version"`
  Rules   []PolicyRule `json:"rules"`
}

type Evaluator struct {
  currentPack PolicyPack
}

func NewEvaluator(pack PolicyPack) *Evaluator {
  return &Evaluator{currentPack: pack}
}

// Evaluate 根据输入的风险评估，结合 Policy Pack 收敛出最终决策
func (e *Evaluator) Evaluate(risk RiskEvaluation) (PolicyDecision, error) {
  if len(risk.Signals) == 0 {
    return PolicyDecision{
      Decision:          Allow,
      PolicyPackVersion: e.currentPack.Version,
      ReasonCode:        "no_risk_signals",
    }, nil
  }

  // 默认决策：放行
  finalDecision := Allow
  finalReason := "pass_default"
  var maxSeverity float64 = -1.0

  // 遍历决策矩阵规则（一期优先采用最高危害度降级收敛法）
  for _, signal := range risk.Signals {
    if !signal.Triggered {
      continue
    }
    for _, rule := range e.currentPack.Rules {
      if signal.DetectorID == rule.RiskCategory && signal.Confidence >= rule.ConfidenceThresh {
        // 权重评估：Block 优先级高于 Redact，Redact 高于 LogOnly
        if e.getDecisionWeight(rule.TargetDecision) > e.getDecisionWeight(finalDecision) || signal.Confidence > maxSeverity {
          finalDecision = rule.TargetDecision
          finalReason = rule.ReasonCode
          maxSeverity = signal.Confidence
        }
      }
    }
  }

  return PolicyDecision{
    Decision:          finalDecision,
    PolicyPackVersion: e.currentPack.Version,
    ReasonCode:        finalReason,
  }, nil
}

func (e *Evaluator) getDecisionWeight(d Decision) int {
  switch d {
  case Block:
    return 100
  case AskConfirmation:
    return 80
  case RedactThenAllow:
    return 60
  case LogOnly:
    return 20
  case Allow:
    return 0
  default:
    return 0
  }
}

```

### 2.3 `tianmu/core/engine.go`

不可旁路的 AI Runtime 引用监视器核心网关。在此预留核心拦截流水线（Pipeline）。

```go
package core

import (
  "crypto/rand"
  "fmt"
  "time"
)

type Engine struct {
  evaluator    *Evaluator
  releaseStage ReleaseStage
}

func NewEngine(stage ReleaseStage, eval *Evaluator) *Engine {
  return &Engine{
    evaluator:    eval,
    releaseStage: stage,
  }
}

func generateTraceID() string {
  b := make([]byte, 16)
  _, _ = rand.Read(b)
  return fmt.Sprintf("%x", b)
}

// MediateInbound 处理输入侧控制面拦截
func (eng *Engine) MediateInbound(
  reqCtx RequestContext,
  idCtx IdentityContext,
  dataCtx DataContext,
  actCtx ActionContext,
  signals []DetectorSignal,
) (*ControlDecisionObject, error) {

  startTime := time.Now()

  // 1. 汇聚外部注入的多路风险信号
  var categories []string
  var maxScore float64
  for _, s := range signals {
    if s.Triggered {
      categories = append(categories, s.DetectorID)
      if s.Confidence > maxScore {
        maxScore = s.Confidence
      }
    }
  }

  riskEval := RiskEvaluation{
    RiskCategories: categories,
    MaxRiskScore:   maxScore,
    Signals:        signals,
  }

  // 2. 调度策略评估器，进行不确定性决策收敛
  policyDec, err := eng.evaluator.Evaluate(riskEval)
  if err != nil {
    return nil, errors.New("failed_to_evaluate_policy")
  }

  // 3. 拦截面特殊控制覆写：如果包含不可逆副作用的 Action，且决策还处于 Allow，强制升级为确认
  if actCtx.SideEffect && policyDec.Decision == Allow {
    policyDec.Decision = AskConfirmation
    policyDec.ReasonCode = "side_effect_action_requires_approval"
  }

  // 4. 自动化组装分布式可审计证据块 (Release Evidence Lite)
  traceID := generateTraceID()
  evidence := ReleaseEvidenceLite{
    EvidenceLevel:     "release_evidence_lite",
    TraceID:           traceID,
    Timestamp:         startTime,
    RegressionPassTag: true, // 默认标记通过，供周日的 CI 回归组件联动改写
  }

  // 5. 组装并实例化最终控制平面对象
  cdo := &ControlDecisionObject{
    ControlID:       fmt.Sprintf("TM-CTL-%d", startTime.UnixNano()),
    Timestamp:       startTime,
    ReleaseStage:    eng.releaseStage,
    RequestContext:  reqCtx,
    IdentityContext: idCtx,
    DataContext:     dataCtx,
    ActionContext:   actCtx,
    RiskEvaluation:  riskEval,
    PolicyDecision:  policyDec,
    ReleaseEvidence: evidence,
  }

  return cdo, nil
}

```

## 三、 Week 1 验证方法 (Verification)

为了确证基础骨架的零缺陷与契约的不变性，我们编写标准的单测文件 `tianmu/core/engine_test.go`。本测试模拟一个典型的 **Chinese User I/O 注入攻击** 场景，验证网关能否正确拦截并输出合规证据。

```go
package core

import (
  "testing"
  "time"
)

func TestTianmuEngine_MediateInbound_BlockInjection(t *testing.T) {
  // 1. 模拟初始化一期工程的 Policy Pack
  mockPack := PolicyPack{
    Version: "v1.0.0-china-personal-default",
    Rules: []PolicyRule{
      {
        RiskCategory:     "prompt_injection",
        ConfidenceThresh: 0.80,
        TargetDecision:   Block,
        ReasonCode:       "cn_malicious_injection_blocked",
      },
      {
        RiskCategory:     "chinese_pii",
        ConfidenceThresh: 0.85,
        TargetDecision:   RedactThenAllow,
        ReasonCode:       "cn_privacy_redaction_triggered",
      },
    },
  }

  eval := NewEvaluator(mockPack)
  engine := NewEngine(PersonalAI, eval)

  // 2. 构造 Inbound 上下文（模拟 Qira 助手的中文交互场景）
  reqCtx := RequestContext{ProductID: "Qira", Language: "zh-CN", InteractionType: "user_io"}
  idCtx := IdentityContext{ActorID: "user_dev_001"}
  dataCtx := DataContext{DataClassification: "public", ContainsPII: false, Source: "user_input", Destination: "model_context"}
  actCtx := ActionContext{ActionType: "generate_response", SideEffect: false}

  // 3. 构造外部注入的中文 Detector 检测到的高置信度风险信号
  mockSignals := []DetectorSignal{
    {
      DetectorID: "prompt_injection",
      Version:    "detector-zh-injection-v1.1",
      Confidence: 0.92, // 高于 0.80 阈值
      Triggered:  true,
    },
  }

  // 4. 执行控制决策
  cdo, err := engine.MediateInbound(reqCtx, idCtx, dataCtx, actCtx, mockSignals)
  if err != nil {
    t.Fatalf("Engine mediation failed: %v", err)
  }

  // 5. 契约一致性断言验证
  if cdo.PolicyDecision.Decision != Block {
    t.Errorf("Expected decision 'Block', got '%s'", cdo.PolicyDecision.Decision)
  }
  if cdo.PolicyDecision.ReasonCode != "cn_malicious_injection_blocked" {
    t.Errorf("Expected reason code 'cn_malicious_injection_blocked', got '%s'", cdo.PolicyDecision.ReasonCode)
  }
  if cdo.ReleaseEvidence.TraceID == "" {
    t.Errorf("Release evidence TraceID must not be empty")
  }

  t.Logf("[Week 1 Success] CDO Generated Successfully. ControlID: %s, TraceID: %s, Decision: %s", 
    cdo.ControlID, cdo.ReleaseEvidence.TraceID, cdo.PolicyDecision.Decision)
}
```
